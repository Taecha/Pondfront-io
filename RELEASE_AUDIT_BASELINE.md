## Database

- Active implementation: SQLite through Node's `node:sqlite` API.
- Default local file: `data/pondfront.db`, using WAL files while running.
- Legacy JSON data file exists at `data/pondfront-db.json`; it is retained and was backed up.
- Persistent tables include users, sessions, OAuth identities/states, player and animal stats, achievements, badges, and match history.
- Release limitation: Render's free filesystem is ephemeral. The current `render.yaml` declares no persistent disk, so deployed SQLite account data may disappear after deploy/restart. A persistent disk or managed external database is required before durable public-account release.

## Existing Feature Baseline

- Server-authoritative grid territory, economy, combat, buildings, abilities, specials, bots, diplomacy, teams, objectives, events, spawn selection, and win conditions.
- Five animals: Duck, Snake, Frog, Turtle, and Carp.
- Four active modes: Classic Elimination, Golden Lily Control, Flood Survival, and Last Nest.

- Desktop and mobile interfaces, shared context actions, minimap, leaderboard, settings, visual presets, and mobile touch controls.
